Questions, answered

Know exactly
what stands guard.

Does Warden replace npm, pnpm, yarn, or Bun?

No. It runs before install and execute commands, then hands cleared packages to the manager you already use. Non-install commands pass through.

Can a generated summary change the verdict?

No. Deterministic rules produce allow, warn, or block. Optional summary rewriting can improve wording, but it cannot change the score, categories, or verdict.

Will a new or unpopular package be blocked?

Not for being new or unpopular alone. Warden uses those facts as context and blocks when stronger evidence, dangerous capabilities, or meaningful signal combinations justify it.

What happens when Warden blocks a package?

The package manager never receives the install. Warden prints the evidence, returns exit code 20, and supports an explicit allow-risky override when a human accepts the risk.

Does it work in pull requests and scripted workflows?

Yes. Warden checks dependency changes against the merge base and provides human, JSON, and workflow annotation reporters using stable exit codes.
